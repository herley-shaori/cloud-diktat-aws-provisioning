Getting started with Google Workspace for your organization

Google Workspace

Your organization's productivity suite

Before configuring SSO to AWS, you need a Google Workspace account. This guide walks you through the registration process.

1 / 6

Prerequisites

What you need before signing up for Google Workspace

🌐

Domain Name

You need a domain name that you own (e.g., yourcompany.com). Purchase from registrars like Namecheap, GoDaddy, or Google Domains.

📧

DNS Access

Access to your domain's DNS settings to verify ownership. You'll add TXT or CNAME records during setup.

💳

Payment Method

Credit card or other payment method for subscription. Google Workspace offers a 14-day free trial.

👤

Admin Contact

Designate who will be the Super Admin. This person manages all Google Workspace settings.

💻

Browser Access

A modern web browser (Chrome, Firefox, Edge, Safari). Google Workspace is fully web-based, no software installation required.

📞

Recovery Phone

A phone number for account recovery and verification. Required for setting up 2-step verification on admin account.

Step 1: Create Users and Groups

Set up users and groups before configuring SSO application

👥 Users

Created users in Google Workspace: admin@cloud-diktat.info and herley@cloud-diktat.info

👥 Groups

Created group Data Scientist (data-scientist@cloud-diktat.info) with 2 members

💡 Why create a group? Only members of the Data Scientist group will be allowed to SSO into AWS. This provides granular access control - not everyone in Google Workspace can access AWS, only those explicitly added to this group.

Step 2: Create SAML Application

Configure Amazon Web Services as a SAML application in Google Workspace

🛠 Web and Mobile Apps

Navigate to Apps > Web and mobile apps, click Add app button, and search for the application name.

Important: Do not create a custom SAML app. Instead, search for "Amazon Web Services" - Google will show a pre-configured template with Platform: Web and Authentication: SAML.

⚙ Application Details

After creating the application, Google Workspace will provide an IdP metadata file. Download this file - you will need to upload it to AWS IAM Identity Center to complete the SAML trust configuration.

User Access

ON for 1 group (Data Scientist) - Only this group can SSO

Service Provider Details

ACS URL and Entity ID - obtain these values from AWS IAM Identity Center

☁ From AWS
SAML Attribute Mapping

Maps Google user attributes to AWS SAML attributes

Auto-provisioning

SCIM enabled - automatically syncs users to AWS
