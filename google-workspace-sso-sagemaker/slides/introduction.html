The Problem: Identity Sprawl

Managing multiple identity systems creates security risks and operational burden

🔑

Multiple Credentials

Data Scientists must remember separate passwords for Google Workspace and AWS Console. Password fatigue leads to weak passwords and security vulnerabilities.

⚠

Offboarding Risk

When employees leave, IT must manually revoke access across multiple systems. Missed accounts become security backdoors.

🕐

Slow Onboarding

New team members wait days for AWS accounts to be provisioned. Manual account creation does not scale with team growth.

📊

Audit Complexity

Compliance audits require matching AWS users to HR records. Separate identity systems make this error-prone and time-consuming.

💰

Hidden Costs

Managing multiple identity systems requires dedicated tools and personnel. Licensing fees and administrative overhead add up quickly.

🚧

Inconsistent Policies

Different systems enforce different password and MFA requirements. Security policies become fragmented and difficult to maintain.

2 / 6

The Solution: Single Sign-On (SSO)

One identity provider, unified access across all platforms

🎯

Single Source of Truth

Google Workspace becomes the authoritative identity provider. One password, one MFA setup, access to everything.

⚡

Automatic Provisioning

SCIM protocol syncs users and groups automatically. Add someone to a Google Group, they instantly get AWS access.

🔒

Instant Deprovisioning

Disable a Google account, AWS access is revoked immediately. No orphaned accounts, no security gaps.

✅

Simplified Compliance

Single audit trail from Google to AWS. Prove who has access and why in minutes, not hours.

💰

Cost Efficiency

No need for separate identity management tools. Reduce administrative overhead and licensing costs.

🚀

Faster Time to Productivity

New team members get AWS access on day one. No tickets, no waiting, just seamless onboarding.

How It Works: The Authentication Flow

SAML 2.0 enables secure identity federation between Google and AWS

1
👤
User Accesses AWS Portal
Data Scientist navigates to https://d-xxxxxxxxxx.awsapps.com/start
2
🔄
Redirect to Google
AWS IAM Identity Center redirects to Google Workspace for authentication
3
🔐
Google Authenticates
User logs in with data-scientist@cloud-diktat.info credentials and completes MFA
4
📄
SAML Assertion Issued
Google creates a signed SAML assertion containing user identity and group memberships
5
✅
AWS Grants Access
AWS validates the assertion and assigns the DataScientist permission set based on group membership

Key Takeaways

Why this architecture works

🔑 Identity Federation

Google Workspace acts as the Identity Provider (IdP). AWS trusts Google to authenticate users via SAML 2.0.

👥 Group-Based Access

Permission sets are assigned to groups, not individuals. Add users to Google Groups to grant AWS access.

📦 Infrastructure as Code

Terraform manages SSO configuration, permission sets, and SageMaker infrastructure. Reproducible and version-controlled.

🔒 Zero Trust Security

Private VPC with no internet gateway. All AWS service access through VPC endpoints.
