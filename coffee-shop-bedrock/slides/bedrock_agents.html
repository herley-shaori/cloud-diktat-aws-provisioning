What is Amazon Bedrock Agents?

Definition

Amazon Bedrock Agents enables you to build autonomous agents that can reason, plan, and execute tasks by orchestrating foundation models, APIs, and data sources.

Key Capabilities

  • Break down complex user requests into steps
  • Collect information through natural conversation
  • Take actions by calling APIs automatically
  • Query knowledge bases for accurate information

How It Works

1 User sends natural language request
2 Agent uses FM to understand intent
3 Agent decides which tools to use
4 Executes actions (Lambda, KB queries)
5 Returns natural language response
Source: AWS Documentation - Automate tasks using AI agents | Amazon Bedrock Agents Product Page

Traditional Approach: Non-Agentic

How Traditional Chatbots Work

// Your application code function handleMessage(message) { if (message.contains("stock") && message.contains("sudirman")) { return callInventoryAPI(); } else if (message.contains("recipe")) { return queryDatabase(); } else { return sendToLLM(message); } }

YOU write the routing logic, YOU handle every intent

Problems

  • Brittle Logic - "how much milk?" won't match "stock"
  • Maintenance Hell - Every new intent = code change
  • No Reasoning - Just pattern matching
  • LLM as Fallback - Not the brain, just backup
  • No Context - Can't combine multiple actions

"Do we need to order milk for kemang?"

This needs: check stock + create PO logic.
In non-agentic: you code ALL combinations!

Agentic Approach: The Solution

How Bedrock Agents Work

You define WHAT tools exist, agent decides WHEN to use them.

// You only define tools (Lambda functions) checkStock(branch_id, item_name) createPurchaseOrder(items, supplier, branch) getQueueLength(branch_id) // Agent automatically: // 1. Understands user intent // 2. Picks the right tool // 3. Extracts parameters // 4. Executes and responds

All These Work Automatically

"Check the milk stock at sudirman"

-> checkStock("sudirman", "milk")

"How much milk do we have?"

-> checkStock("sudirman", "milk")

"Is there enough milk at main branch?"

-> checkStock("sudirman", "milk")

"Sudirman inventory for milk please"

-> checkStock("sudirman", "milk")

Agentic vs Non-Agentic: Comparison

Capability Non-Agentic (Traditional) Agentic (Bedrock Agents)
Intent Recognition Hardcoded keywords, regex patterns LLM understands semantic meaning
Tool Selection IF-ELSE routing logic in your code Agent decides dynamically based on context
Parameter Extraction Manual parsing (regex, NLP library) Agent extracts from natural language
Adding New Features Code changes, redeploy app Add new tool definition, done
Multi-step Tasks Code every combination manually Agent chains tools automatically
Response Format Template strings Natural language generation
Maintenance High - every edge case = code Low - just update tool definitions

Building Bedrock Agents: Components

1. Foundation Model

Claude 3 Haiku for agent reasoning. Understands user intent and orchestrates actions.

2. Agent Instructions

System prompt defining persona, capabilities, and behavior guidelines.

3. Action Groups

Lambda functions that execute real actions (check inventory, create orders).

4. Knowledge Base

RAG system with S3 + OpenSearch for recipes, SOPs, FAQ documents.

5. Guardrails

Content filters, topic blocks, PII protection for safe responses.

6. Agent Alias

Version pointer for production deployment. Required for InvokeAgent API.

Request Flow: Agentic Action

User: "Check the milk stock at sudirman branch"

Agent Reasoning Process

1. Understanding: User wants to check inventory 2. Location: Sudirman branch identified 3. Item: Milk specifically 4. Tool Selection: I have checkStock function 5. Parameters: - branch_id = "sudirman" ✓ - item_name = "milk" ✓ 6. Decision: Call checkStock with these params

Lambda Invocation

{ "function": "checkStock", "parameters": [ {"name": "branch_id", "value": "sudirman"}, {"name": "item_name", "value": "milk"} ] }

Lambda Response

{ "status": "success", "branch": "sudirman", "item": "milk", "quantity": 15, "unit": "liters", "stock_status": "OK", "needs_reorder": false }

Final Response

"According to the inventory check, the Sudirman branch currently has 15 liters of milk in stock. The minimum stock level is 5 liters, and the current stock is above minimum, so no reordering is needed."

Key Takeaways

Why Agentic?

  • LLM is the Brain - Not just a fallback, it orchestrates everything
  • No Routing Code - Agent decides which tool to use
  • Natural Language In/Out - Understands variations automatically
  • Extensible - Add tools without code logic changes
  • Multi-step Capable - Can chain actions intelligently

Coffee Shop Agent Capabilities

  • checkStock - Check inventory at any branch
  • createPurchaseOrder - Order from suppliers
  • getQueueLength - Real-time wait times
  • Knowledge Base - Recipes, SOPs, FAQ
